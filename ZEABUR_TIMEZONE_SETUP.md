# Zeabur 環境變數設定指南

## 設定時區以修復「今日已複習」不同步問題

### 問題說明
Zeabur 伺服器預設使用 UTC 時區（比台灣慢 8 小時），導致：
- 台灣時間已經是新的一天，但雲端還停留在昨天
- 「今日已複習」數量與本地端不同步

### 解決方法：設定時區環境變數

#### 步驟 1：登入 Zeabur
前往 https://zeabur.com/ 並登入您的帳號

#### 步驟 2：選擇專案
在 Dashboard 中找到並點擊 `AICowLaw` 專案

#### 步驟 3：設定環境變數
1. 點擊專案中的服務（Service）
2. 找到「Environment Variables」或「環境變數」標籤
3. 點擊「Add Variable」或「新增變數」

#### 步驟 4：新增時區變數
- **Variable Name（變數名稱）**: `TZ`
- **Value（值）**: `Asia/Taipei`

#### 步驟 5：儲存並重新部署
1. 點擊「Save」或「儲存」
2. Zeabur 會自動重新部署
3. 等待 2-3 分鐘讓部署完成

### 驗證修復
部署完成後：
1. 訪問 https://aicowlaw.zeabur.app/
2. 登入系統
3. 檢查「今日已複習」數量
4. 應該與本地端一致了！

### 其他可用的時區值
- `Asia/Taipei` - 台北（推薦）
- `Asia/Shanghai` - 上海（同樣是 UTC+8）
- `UTC` - 協調世界時（預設值）

### 注意事項
- 設定後需要重新部署才會生效
- 此設定不會影響資料庫中已儲存的時間
- 只影響程式執行時的時間判斷

### 如果還是不同步
1. 確認環境變數已正確設定
2. 確認已重新部署
3. 清除瀏覽器快取後重新載入
4. 檢查本地端和雲端是否使用同一個 Airtable 資料庫
